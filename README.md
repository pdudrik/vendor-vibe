# vendor-vibe
Web application dedicated for managing and monitoring company expenses - vendors (B2B contracts).

## Setup

1. Create venv.
2. Install requirements.txt.
3. Create .env file and insert enter hosts and secret key.

```
SECRET_KEY=""
HOSTS="127.0.0.1,localhost"
TIME_ZONE="Europe/Bratislava"
LANGUAGE="en-us"
```


## Date and time
Time zone set in `.env` file (Europe/Bratislava).<br>
If time zone is set (which is; not using UTC NOW), don't use naive `datetime.now()` or `datetime.today()` because Django will crash. Use instead Django utils such as:
```
from django.utils import timezone

current_time = timezone.now()
today = timezone.localdate()
```

## Models
Format rules:<br>
<b>Date and time:</b> ISO 8601 (at least for ui)<br>
`yyyy-MM-dd HH:mm:ss` -> `2026-04-19 09:48:12`<br>
<b>Currency:</b> ISO 4217<br>
`Euro` -> `EUR`<br>
<b>Country code:</b> ISO 3166-1 aplha-2<br>
`Slovakia` -> `SK`<br>
<br>

`Vendor`
- `company` - int, FK to Company model, not null
- `id` - int, pk, unique
- `name` - str, not null
- `service_category` - str, not null (e.g. marketing, legal, finance)
- `status` - boolean (active/inactive)

`Address`
- `id` - int, pk, unique
- `vendor_id` = int, FK to Vendor model, not null
- `street` - str, not null (e.g. 1035/22 Namestie Osloboditelov, Apt 9C)
- `street_opt` - str (some more info if needed)
- `city` - str, not null
- `state_provinence` - str (e.g. Kosicky kraj)
- `postal_code` - str, not null (some countries use letters in postal code e.g. Canada: K1A 0B1)
- `country_code` - str, nor DBt null
- `address_type` - str, not null (choice - "Billing", "Shipping", "Headquarters"; use shortcuts maybe)

`Contact`
- `id` - int, pk, unique
- `vendor_id` - int, FK to Vendor model, not null
- `first_name` - str
- `last_name` - str, not null
- `middle_name` - str
- `number` - str, not null or email (whole number +421902123456)
- `email` - str, not null or number

`Contract`
- `id` - int, pk, unique
- `vendor_id` - int, FK to Vendor model, not null
- `name` - str, not null
- `description` - str, simple or detailed description of contract

`Invoice`
- `id` - int, pk, unique
- `contcract_id` - int, FK to Concract model, not null
- `invoice_id` - str, unique, not null (unique generated by invoice provider)
- `ammount` - str, using django money field (stores ammount and currency in certain ISO format)
- `issue_date` - date, not null
- `due_date` - date, not null
- `status` - str, not null (choice - "Draft", "Pending", "Paid", "Overdue"; use shortcuts maybe)
- `invoice_path` - str (probably URL path to file stored on server)
